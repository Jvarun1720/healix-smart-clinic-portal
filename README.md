Welcome to **HEALIX**, a full-stack, cross-platform health-tech application built to connect patients and medical practitioners in real time. 

As a **3rd-year Electrical and Electronics Engineering (EEE) student**, I built this project to bridge the gap between core engineering logic and modern full-stack cloud software architecture. Instead of just creating a simple static website, I engineered a dynamic ecosystem handling complex user flows and database relations entirely from scratch.

## 💡 What the Application Does

HEALIX operates as a dual-channel platform that seamlessly splits workflows based on user roles:

* Centralized Landing Page (`/`): A custom, glassmorphic welcome screen that acts as a router guiding users to their respective secure environments.
* The Patient Portal (`/patient`): Allows patients to log in, discover registered medical specialists across departments, select open slots, and push dynamic appointment requests directly to the cloud.
* The Practitioner Hub (`/doctor`): An operational dashboard for doctors to monitor incoming patient logs, manage daily appointment queues, and interactively update states (Pending / Confirmed / Cancelled).

---

## 🛠️ The Tech Stack (What Code was Used)

* Frontend Engine: `Next.js 15+` (React-based framework) using the modern **App Router** system for fast, optimized page switching.
* Compiler Stack: Powered by **Turbopack** for instantaneous local page hydration and performance.
* Styling Framework:`Tailwind CSS` for a premium, clean, custom responsive aesthetic with modern gradients.
* Programming Language: `TypeScript` / `JavaScript` ensuring type safety and robust data validation.
* Backend Database: `Supabase` running a cloud-hosted relational **PostgreSQL** engine.

## 🔧 Core Engineering Accomplishments & Skills Learned

Through this build, I didn't just learn syntax—I learned real-world software engineering concepts:

* Relational Database Design (SQL): Structured matching tables with **Foreign Key Constraints** to link user bookings securely to authenticated doctor identities, preventing database errors or orphaned records.
* Database Fixes (Cascading Actions): Implemented advanced database commands (`DROP TABLE CASCADE`) to clean up complex dependency policies safely.
* Progressive Web App (PWA) Architecture: Wrote background configuration manifests (`manifest.json`) and active **Service Workers (`sw.js`)** allowing the entire web system to bypass browser boundaries and become **natively installable** onto a computer desktop or mobile phone home screen.
* Asynchronous Data Handling: Managed real-time data fetching, session token handling (`supabase.auth.getUser()`), and state-driven form submissions.
